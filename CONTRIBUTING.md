# Contributing to Lantern

Thank you for helping make agentic life organization approachable to more people.

## Before changing anything

- Keep Lantern useful for nontechnical users.
- Keep personal examples generic and synthetic.
- Do not add names, companies, schools, email addresses, usernames, phone numbers, addresses, credentials, or local computer paths.
- Do not add required external services, analytics, or telemetry.
- Preserve exact-action approval for consequential actions.
- Verify current product and model claims against official documentation.

## Make a change

1. Create a focused branch.
2. Add or update a failing test before behavior changes.
3. Make the smallest change that satisfies the test.
4. Run the complete test, validator, packaging, and privacy commands documented in the implementation plan.
5. Describe user impact, validation, and any account-dependent behavior that remains unverified.

## Writing style

Use plain language, warmth, and honest limits. Explain technical terms only when the user needs them. Do not promise direct cross-Project messaging, automatic file updates, or app actions the platform does not expose.

## Pull requests

Keep unrelated cleanup out of the change. Never include private user material in issues, commits, tests, screenshots, or pull requests.
