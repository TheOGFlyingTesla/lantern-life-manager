# Optional Assisted Setup

[Back to the installation guide](INSTALL.md)

Most people should follow the normal four-step installation. No Skill or developer tooling is required.

If Codex is already available, the user may give it the newest release link and say:

> Download `lantern-desktop-v2.0.0.zip` anonymously, extract it to the writable location I choose, and verify the complete Lantern folder. Do not install GitHub, Git, Python, Xcode, Apple developer tools, or a Skill. Do not create cloud Projects or one Project per domain. When the folder is ready, give me the single desktop step for adding it as one local Project.

Codex may prepare the folder, but registration in the desktop Projects view can still require one folder-picker action. It must not claim the local Project exists until the user or an available app control verifies it.

If Codex itself requests a platform prerequisite for an unrelated operation, stop and use the normal ZIP extraction path instead. Lantern does not require that prerequisite.
